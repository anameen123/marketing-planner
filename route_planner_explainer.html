🗺️ Route Planner — How clinics get bucketed

Member sets Start and End + how many stops they want in between (here: 5). The algorithm fans out perpendicular corridors along the straight line and assigns each clinic to a level.

What the picture shows: 5 stops requested between Start and End. The straight line is split into 5 equal segments of length d. Each segment becomes a rectangle extended width mi on each side perpendicular to the line. Any clinic/MVA pin falling inside that rectangle becomes a candidate for that level. Pins outside are ignored unless the user expands width.

🎨 Pin colors

Green · > 3 weeks since last visit (or never) — TOP PRIORITY
Orange · 2–3 weeks since last visit — secondary
Red · < 2 weeks ago — EXCLUDED from recommendations
Purple S/E · Start & End pins

📐 The math

L = straight-line distance(S → E)
d = L / N (here N = 5)
width = user-tunable (default 1 mi/side)
in_level_i(pin) =
  projection onto line ∈ [i·d, (i+1)·d]
  AND ⊥ distance from line < width

🎛️ Member controls

  • Set start / end via list pick OR double-click pin on map
  • Number of stops (changes N, keeps the same level structure)
  • Corridor width slider — expand to catch more pins per rect
  • Pick 0–N candidates per level (some levels can be empty)

🔑 API keys needed?

No. The algorithm is pure client-side math on coordinates we already have. Google Maps / Gemini are only needed if you want fancy driving routes or text recommendations from an LLM — your geometry approach doesn't need them.
